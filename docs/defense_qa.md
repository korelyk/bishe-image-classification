# 答辩常见问题与参考回答

## 1. 为什么题目叫“图片分类系统”，但你强调的是道路目标七分类？

因为课题来源于自动驾驶道路场景，业务目标并不是通用图片管理，而是识别道路图片中与驾驶感知最相关的目标类别。为了兼顾课题名称与实际实现，我把系统做成“面向道路场景的图片分类系统”，即把图片最终归入七类交通相关目标中。

## 2. 为什么没有完全按开题里的小程序、Spring Boot、Vue3、MySQL 来做？

这是一个有意识的工程化折中。毕业设计最重要的是做出真正可运行、可部署、可演示、可答辩的完整成品。若严格按多端分离路线开发，联调和部署成本更高，不利于在有限周期内稳定交付。因此我采用 FastAPI 一体化方案，把重点放在深度学习推理、前后台页面、数据库留痕和真实部署上。

## 3. 你的深度学习模型是什么？

部署版系统使用 ONNX Runtime 加载 MobileNetV2 预训练模型。模型先输出 ImageNet 分类结果，再通过映射规则转换为七类道路目标。这样既保留了深度学习推理能力，又显著降低了部署体积和资源消耗。

## 4. 为什么选 ONNX Runtime + MobileNetV2？

主要有三个原因：第一，模型轻量，适合服务器资源受限场景；第二，ONNX Runtime 部署简单、CPU 推理友好；第三，能保证系统真的上线运行，而不是停留在论文设计层面。

## 5. 你的系统支持哪些类别？

系统支持七类道路相关目标：行人、自行车、汽车、摩托车、公交车、卡车、交通标志/信号灯。

## 6. 如果模型输出不是这七类怎么办？

我采用了 ImageNet 标签映射策略。系统会读取 Top-5 候选标签，只要其中有能映射到七类目标的标签，就将其作为最终结果；如果没有，则归为“未识别”。这种设计提高了系统稳定性。

## 7. 你的系统有哪些主要模块？

主要包括：
1. 用户端上传与结果展示模块；
2. 管理端统计与记录查看模块；
3. 深度学习推理模块；
4. SQLite 数据存储模块；
5. systemd 部署与运维模块。

## 8. 数据库存了什么？

数据库中存储了文件名、原图路径、预测类别、中文标签、置信度、模型模式、原始候选标签 JSON 和创建时间，便于历史追溯和后台统计。

## 9. 你的系统如何部署？

我将系统部署到 Linux 服务器独立目录中，通过 Python 虚拟环境安装依赖，使用 systemd 托管 uvicorn 服务，并开放独立端口对外提供访问。

## 10. 系统有哪些不足？

第一，当前使用的是通用预训练模型和标签映射，还不是针对道路七类专门训练的模型；第二，目前属于整图分类，无法像检测模型那样给出精确目标框；第三，数据库使用 SQLite，更适合中小规模演示场景。后续可进一步替换为自训练 YOLO/ResNet，并接入 MySQL 与前后端分离架构。
