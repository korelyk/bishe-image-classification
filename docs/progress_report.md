# 毕设任务落实说明

## 1. 题目落实

题目：**基于深度学习的图片分类系统**  
方向：**自动驾驶道路目标七分类**

系统已围绕道路场景图片完成可运行的 Web 成品，满足“图片上传、分类结果展示、后台/数据管理、深度学习模型、论文正文不少于 8000 字、服务器部署验证”的核心要求。

## 2. 功能落实情况

### 2.1 图片上传与分类结果展示

**落实情况：** 已完成用户端首页，可上传 JPG/PNG/WEBP 图片，并展示主分类、置信度、推理模式、Top-5 候选标签及最近历史记录。

### 2.2 后台与数据管理

**落实情况：** 已完成 `/admin` 管理页面，可查看总识别次数、最近识别时间、类别分布、最近 10 条识别记录，并支持管理员口令保护。

### 2.3 深度学习模型接入

**落实情况：** 系统采用 ONNX Runtime 加载 MobileNetV2 预训练模型执行图片分类，通过 ImageNet 标签映射到七类道路目标。这一方案属于轻量化深度学习部署，实现了真正可运行、可上线、可演示的推理链路。

### 2.4 数据留痕

**落实情况：** 使用 SQLite 持久化保存文件名、原图路径、预测类别、置信度、模型模式、原始候选标签 JSON 和创建时间。

### 2.5 部署与验证

**落实情况：** 提供远程部署脚本、systemd 服务、健康检查接口和 smoke test 脚本，可在服务器中独立目录、独立端口运行，不影响现有服务。

## 3. 与任务书/开题路线的差异说明

任务书/开题材料偏向“小程序 + Spring Boot + Vue3 + MySQL + 深度学习训练框架”的理想路线。实际交付版本采用：

- 前后端：FastAPI + Jinja2 + 原生 JavaScript
- 模型部署：ONNX Runtime + MobileNetV2
- 数据库：SQLite
- 部署：systemd + uvicorn

**调整原因：**
1. 毕设周期有限，优先保证真实可交付；
2. 指定服务器磁盘空间有限，不适合安装重量级 CUDA 依赖；
3. 轻量方案更易完成真实部署、调试和答辩演示；
4. 仍保留深度学习分类、前后台页面、数据库和部署链路，满足毕业设计核心要求。
