# 任务书 / 开题要求提炼

## 一、题目

**基于深度学习的图片分类系统**

## 二、业务方向

- 面向自动驾驶道路场景
- 围绕七类道路相关目标进行分类
- 需要形成可展示、可答辩的完整本科毕设成品

## 三、核心功能要求

1. 支持图片上传
2. 支持分类结果展示
3. 支持后台管理或数据管理
4. 必须体现深度学习模型能力
5. 论文正文不少于 8000 字
6. 最终系统需要可运行、可部署、可验证

## 四、工程化落地策略

为保证项目能在毕设周期内真正完成并上线，交付版采用以下实现策略：

- 前后端：FastAPI + Jinja2 + 原生 JavaScript
- 深度学习推理：ONNX Runtime + MobileNetV2 预训练模型
- 类别统一：ImageNet 标签映射到七类道路目标
- 数据存储：SQLite
- 部署方式：systemd + uvicorn

## 五、满足要求的对应关系

- “图片上传” → 首页上传表单与 `/api/classify`
- “分类结果展示” → 首页结果卡片、置信度、候选标签明细
- “后台/数据管理” → `/admin` 页面、统计接口、SQLite 记录
- “深度学习模型” → ONNX Runtime 加载预训练 CNN 模型推理
- “论文不少于 8000 字” → `docs/thesis_full.md`
- “部署验证” → `scripts/deploy_remote.sh`、`scripts/smoke_test.sh`

## 六、说明

开题方案中的技术路线可作为参考，但交付版优先保证真实可运行、可部署和可答辩，因此采用了更轻量、更稳定的深度学习工程化方案。
