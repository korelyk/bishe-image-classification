# 《基于深度学习的图片分类系统》答辩 PPT 提纲

## 1. 课题背景与意义
- 自动驾驶与智慧交通对图像理解能力的需求不断增强
- 道路场景识别是环境感知的重要基础
- 本课题目标：实现一套可运行、可部署、可演示的图片分类系统

## 2. 课题目标
- 面向道路交通场景图片完成智能识别
- 将结果统一映射为七类目标
- 实现用户端上传、结果展示、后台统计、历史留痕与服务器部署

## 3. 需求分析
- 输入：道路场景图片
- 输出：七类道路目标之一或“未识别”
- 功能：上传、分类、候选标签展示、记录查询、后台统计
- 非功能：可运行、可部署、可答辩展示

## 4. 总体技术方案
- 前后端：FastAPI + Jinja2 + JavaScript
- 推理引擎：ONNX Runtime + MobileNetV2
- 数据存储：SQLite
- 部署方式：Python 虚拟环境 + Uvicorn

## 5. 系统架构设计
- 前端展示层
- 后端接口层
- 模型推理层
- 数据持久化层
- 部署运行层

## 6. 核心实现
- 图片上传与保存
- ONNX 模型加载与预处理
- Top-5 分类标签解析
- ImageNet 标签到七类道路目标的映射
- 历史记录入库与后台统计

## 7. 系统功能展示
- 首页上传识别
- 识别结果展示
- 最近记录展示
- 后台统计界面
- 健康检查接口

## 8. 实验与验证
- 健康检查通过
- 上传识别接口通过
- 后台统计接口通过
- 演示样例：自行车、公交车、汽车识别成功

## 9. 工程化优化说明
- 原计划采用小程序 + Spring Boot + Vue3 + MySQL
- 实际交付版采用一体化 Python 方案
- 原因：降低联调和部署复杂度，保证系统真实可交付

## 10. 项目亮点
- 深度学习能力真实上线可运行
- 前台、后台、数据库、部署链路完整
- 演示效果直观，适合本科毕设答辩

## 11. 不足与改进方向
- 目前仍以预训练模型 + 标签映射为主
- 尚未针对道路数据集做专门微调
- 后续可升级为 YOLO/ResNet 微调、多端分离、MySQL 存储

## 12. 总结
- 已完成一套能运行、能部署、能答辩展示的毕业设计成品
