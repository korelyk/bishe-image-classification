# 实验结果与演示样例摘要

## 一、运行环境
- 操作系统：Linux
- Python：3.9
- 推理框架：ONNX Runtime 1.19.2
- Web 框架：FastAPI
- 数据存储：SQLite

## 二、功能验证结果

### 1. 健康检查接口
- 地址：`/api/health`
- 结果：返回 `status=ok`
- 结论：服务运行正常

### 2. 图片上传识别接口
- 地址：`/api/classify`
- 结果：能够接收图片并返回分类结果、置信度、候选标签
- 结论：核心功能正常

### 3. 历史记录接口
- 地址：`/api/history`
- 结果：能够查询最近识别记录
- 结论：数据库写入与查询正常

### 4. 后台统计接口
- 地址：`/api/admin/stats`
- 结果：能够返回总次数、类别分布、最近记录
- 结论：后台统计模块正常

## 三、推荐演示样例

### 1. 自行车样例
- 文件：`data/demo/bike_demo.jpg`
- 识别结果：自行车
- 说明：Top-1 标签为 `mountain bike`

### 2. 公交车样例
- 文件：`data/demo/bus_demo.jpg`

### 3. 汽车样例
- 文件：`data/demo/car_demo.jpg`
- 识别结果：汽车

## 四、实验结论
- 系统已具备稳定的上传、识别、记录和后台管理功能
- 对适合的道路交通图片样例能够给出较合理的识别结果
- 满足本科毕业设计“系统可运行、可部署、可展示”的要求

## 五、不足与改进方向
- 当前主要依赖预训练模型与标签映射，未做专门微调
- 对复杂场景和非典型图片仍可能输出“未识别”
- 后续可结合专用道路数据集进行微调训练和性能优化
