# 现场演示话术稿

## 开场（30 秒）
各位老师好，我的毕业设计题目是《基于深度学习的图片分类系统》。本系统面向道路交通场景图片，能够对上传图片进行深度学习推理，并将结果统一映射为七类道路目标，同时提供历史记录和后台统计功能。

## 第一步：介绍首页（30 秒）
这里是系统首页。用户可以上传本地图片，系统会调用部署在服务器上的模型完成识别。首页同时展示系统支持的七类目标，以及最近识别记录，方便演示整体功能。

## 第二步：演示识别（1 分钟）
我先上传一张自行车图片。系统返回“自行车”，并展示 Top-5 候选标签与置信度。接着上传公交车样例，系统识别为“公交车”。再上传汽车样例，系统识别为“汽车”。

## 第三步：介绍模型逻辑（40 秒）
部署版系统使用 ONNX Runtime 加载 MobileNetV2 预训练模型，先得到 ImageNet 分类结果，再通过标签映射策略统一到七类道路目标。这种方案的优点是部署轻量、CPU 友好，适合毕业设计答辩场景下稳定运行。

## 第四步：展示后台（40 秒）
进入后台页面后，可以查看总识别次数、最近识别时间、类别分布以及最近 10 条记录。这样可以体现系统不仅能识别，也具备数据留痕和管理能力。

## 第五步：说明工程化取舍（40 秒）
原开题中的理想路线是小程序、Spring Boot、Vue3 和 MySQL。考虑到毕业设计周期和部署复杂度，我最终采用了 FastAPI 一体化方案，重点保证深度学习推理、页面展示、数据库留痕和真实部署全部打通。

## 结束语（20 秒）
总体来说，本项目完成了从需求分析、系统实现到服务器部署和答辩文档准备的完整闭环，能够满足本科毕业设计的展示与答辩要求。
